### 1\. Введение <a name="1"></a>

Данное веб-приложение представляет собой программный инструмент, предназначенный для автоматизированной генерации графических идентификаторов различных типов: линейных штрихкодов (EAN), двухмерных матричных кодов (Data Matrix) и матричных кодов быстрого отклика (QR-код). Основная задача сервиса — обеспечить пользователю удобный интерфейс для преобразования текстовой или цифровой информации в стандартизированные графические форматы, пригодные для печати, логистического учета и маркетинговых целей.

### 2\. Требования пользователя <a name="2"></a>

#### 2.1. Программные интерфейсы <a name="2.1"></a>

Для реализации проекта используется современный стек веб-разработки на базе Node.js. Взаимодействие между компонентами системы организовано следующим образом:
Серверная часть (Backend): Построена на фреймворке Express. Используется формат модулей ESM (import/export), что обеспечивает соответствие современным стандартам JavaScript.
Генерация кодов: Для создания графических идентификаторов интегрированы специализированные библиотеки:
qrcode — для формирования QR-кодов.
bwip-js — для генерации сложных форматов (EAN-13, EAN-8, Data Matrix).
Графический интерфейс (Frontend): Реализован на «чистом» стеке (Vanilla HTML/CSS/JS) без использования тяжеловесных фреймворков и сборщиков, что обеспечивает максимальную скорость загрузки и простоту поддержки.
Связь: Взаимодействие фронтенда с API сервера осуществляется через Fetch API. Для обеспечения возможности внешних запросов подключен модуль

### 3\. Системные требования <a name="3"></a>

#### 3.1. Функциональные требования <a name="3.1"></a>

Пользователю предоставлены следующие возможности:

1. Выбор типа генерируемого штрихкода: QR-code, data matrix или EAN.
2. Выбор типа кодируемой информации для QR-code и data matrix.
3. Выбор типа EAN: EAN-13 или EAN-8.
4. Выбор формата получаемого изображения: PNG или SVG.
5. Скачивание полученного изображения.

#### 3.2. Нефункциональные требования <a name="3.2"></a>

##### 3.2.1 Атрибуты качества <a name="3.2.1"></a>

* Приложение должно проверять корректность ввода данных для генерации штрихкода.
* Должна быть возможность из.
* Игра имеет низкие системные требования так как не задействует большое количство ресурсов компьютера.

##### 3.2.2 Ограничения <a name="3.2.2"></a>

Запуск приложения производится под любой операционной системой на которой установлен JVM.

### 4\. Описание приложения <a name="4"></a>

##### 4.1 Цель приложения <a name="4.1"></a>

Данное приложение предназначено для генерации штрихкодов.

##### 4.2 Руководство пользователя <a name="4.2"></a>

Для работы с генератором пользователю необходимо выполнить следующие действия:
Выбор стандарта: В выпадающем списке интерфейса выберите нужный тип кода: EAN-13 или EAN-8 (для товарной маркировки), Data Matrix (для двумерного кодирования) или QR-код (для ссылок и текстов).
Ввод данных: Введите информацию в текстовое поле. Система автоматически подстроит формат под выбранный стандарт (например, для EAN-13 требуется ввод 12 или 13 цифр).
Генерация: Нажмите кнопку «Создать». Благодаря серверной обработке на базе Node.js, изображение кода мгновенно отобразится в окне предпросмотра.
Сохранение: Чтобы сохранить полученный код, нажмите кнопку «Скачать» или воспользуйтесь правой кнопкой мыши на изображении для выбора функции «Сохранить как».
Контроль: В случае ввода некорректных данных (например, букв в цифровой код EAN), приложение выведет уведомление об ошибке, полученное от сервера.

### 5\. Аналоги <a name="5"></a>

* [Online Barcode Generator](https://barcode.tec-it.com/ru) — мощный профессиональный ресурс, поддерживающий более 100 типов кодов, включая почтовые и медицинские стандарты, однако требующий оплаты за скачивание изображений в высоком качестве без водяных знаков.

* [Zint Barcode Generator](https://zint.org.uk/) — программное обеспечение с открытым исходным кодом, которое служит эталоном точности кодирования, но требует установки на компьютер и не имеет удобного веб-интерфейса для рядового пользователя.
